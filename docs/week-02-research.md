# Week 2 — Research & Integrative Activity

## Overview
This week focused on researching idea validation techniques and planning strategies to improve the Springs Explorer project.

---

## Business Case & Idea Validation

### Case Study Insights
After reviewing Google case studies, I learned that testing small features early and gathering feedback helps reduce wasted development time. This approach applies to my project by validating search filters and spring detail views before adding advanced features.

---

### A/B Testing Takeaways
A/B testing can be used to compare different UI layouts or call-to-action placements. For Springs Explorer, this could help determine whether users engage more with card-style layouts or list-style layouts.

---

### Use Case Planning
Use cases help define expected user behavior. Example use case for my app:

- User opens homepage
- User browses spring list
- User selects spring details
- User views activities and amenities

This ensures the app flow remains focused and usable.

---

## Agile Planning & Workflow

Using GitHub Issues and Milestones helps break work into manageable tasks and track weekly progress. This allows better organization and visibility into project development.

---

## Tools & Strategy For Project

Planned tools and techniques:
- API data validation
- Frontend UI testing
- Use-case testing for navigation flow
- GitHub Projects for sprint planning

---

## Reflection

This research helped reinforce the importance of validating ideas early and maintaining organized development workflows to avoid rework later.
